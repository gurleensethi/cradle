# Project: Cradle

## Overview

Cradle is a Go-based CLI tool that helps developers manage their local projects efficiently. It provides both a TUI (Terminal User Interface) and command-line commands for project management.

## Project Conventions

### Golang Instructions
- Follow idiomatic go, not idiotic go!
- Every function should be documented properly and concisely with what it does
- Focus on documenting the behaviour of the function instead of implementation details
- Documentation should be meaningful to the developer reading it
- No package doc required
- Don't document simple and obvious code and functions

### Code Style
- Use clear, descriptive function and variable names
- Keep functions focused and single-purpose
- Follow Go community standards and best practices

### Testing
- Write tests for new functionality
- Maintain existing test coverage
- Use table-driven tests where appropriate

### Documentation
- Update README.md when adding new features
- Keep documentation in sync with code changes
- Use the code-docs-sync skill to maintain documentation quality

## Skills Available
- `code-docs-sync`: Ensure code documentation is accurate, coherent, and synchronized with implementation
